Gap-check vs approved spec/backend — fully covered, one small backend addition surfaced & folded into the spec: list + stat cards → distributors + distributorStats; status badges/actions → setStatus (built); Overview details/location → existing distributors.region/address + shop_users (no new backend); set-limit → setCreditLimit (T7); record repayment/adjustment + Add/Subtract → record_credit_movement (built) via recordCreditMovement (T8); Credit ledger with actor column → listCreditMovements (actor already stored); Activity tab → new listDistributorActivity over the existing F0 audit_log (no new table); Orders tab = deferred placeholder (Orders sub-project). Purchase draws intentionally not a UI action.
